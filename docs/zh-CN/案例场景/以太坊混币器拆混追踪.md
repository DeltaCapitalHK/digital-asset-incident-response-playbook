# 以太坊 ETH 被盗后进入混币器的追踪场景

本页面适用于 ETH 或 ERC20 资产被盗后进入混币器、隐私工具或固定面额池的情况。

混币器案件不能简单写成“可以拆出来”。更专业的做法是说明分析方法、置信度和限制条件。

## 脱敏案例摘要

一名用户的钱包在钓鱼签名或授权攻击后被快速转走约 119.78 ETH。攻击者先将 ETH 转入控制地址，再分批存入固定面额混币器，试图切断链上关联。随后，混币器提款侧出现多个候选地址，部分资金继续经过剥离、归集，并出现疑似进入中心化交易所的迹象。

该类案例的难点在于：ETH 是原生币，没有 USDT 这类发行方冻结机制；一旦资金离开可协查平台，链上单方面冻结空间非常有限。因此，工作重点是尽快还原混币前路径、建立混币后候选关系、标注置信度，并在资金进入交易所或其他可协查节点前准备材料。

公开版案例链路可概括为以下方向。具体候选地址筛选、置信度模型和归因细节不在公开材料中展开：

```text
受害钱包
  -> 可疑归集地址
  -> 隐私工具或混合路径
  -> 候选下游路径
  -> 疑似平台或其他可协查节点
```

这个案例适合说明“混币器拆混”不是玄学，也不是绝对结论，而是基于时间窗、金额面额、gas 来源、地址复用、后续归集和交易行为做置信度分析。公开材料不展示真实混币器、真实地址、交易哈希和交易所账号。

## 典型场景

钱包遭遇钓鱼攻击、私钥泄露或恶意授权后，ETH 被快速转入攻击者地址，再分批进入混币器或隐私工具。攻击者试图通过混合池切断链上关联，随后再把资金分散、兑换或转入交易所。

## 需要准备的材料

- 被盗钱包地址
- 首笔被盗交易哈希
- 被盗金额和时间
- 攻击者初始归集地址
- 进入混币器的交易哈希
- 混币器前后的金额、时间和地址
- 可疑提款地址
- 后续交易所充值迹象
- 钓鱼网站、签名记录、钱包截图

## 分析方法

### 1. 混币前路径重建

先还原资金从受害钱包到攻击者控制地址，再到混币器存款地址的路径。混币前路径通常比混币后路径更确定。

### 2. 时间窗分析

观察存入和提出的时间窗口。如果多个提款在存款后相近时间出现，可能形成候选集合，但不能单独作为确定证据。

### 3. 金额和固定面额

部分混币器使用固定面额池。分析时可比较存款和提款面额，但金额一致不代表必然属于同一控制人。

### 4. 行为特征和地址关联

如果候选地址在后续行为、地址关联或归集方向上与前序路径存在可解释联系，可提高分析置信度。

### 5. 置信度标注

混币器分析应明确标注：

- 高置信度
- 中置信度
- 低置信度
- 待平台或执法机关核实

不要把概率判断写成绝对结论。

## 从该案例拆出的材料方向

```text
钱包被盗或授权钓鱼经过
关键交易记录
混合路径摘要
候选下游方向和置信度说明
交易所/执法/法律程序辅助材料
```

## 原生币和稳定币的区别

ETH 是原生币，没有 USDT 这类发行方冻结机制。如果资金已经离开可协查平台，单纯链上冻结通常不可行。因此时间窗口、交易所协查和证据提交速度非常重要。

## 可交付材料

- 混币前资金路径
- 候选提款地址列表
- 置信度说明
- 交易所落点说明
- 报案辅助材料
- 交易所协查材料

## 注意事项

混币器场景复杂，公开材料不应承诺“必然拆混”或“必然追回”。专业表达应聚焦证据、路径、候选关系和置信度。
