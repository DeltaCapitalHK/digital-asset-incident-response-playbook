# 以太坊隐私化路径资金流转场景

本页面适用于 ETH 或 ERC20 资产被盗后进入隐私化或混淆路径的情况。

公开材料只说明事件类型、材料方向和限制条件，不展示内部分析方法。

## 脱敏案例摘要

一名用户的钱包在钓鱼签名或授权攻击后被快速转走约 119.78 ETH。攻击者随后将资金导入隐私化或混淆路径，后续出现若干可继续整理的方向，并疑似接近可协查平台环境。

该类案例的难点在于：ETH 是原生币，没有 USDT 这类发行方冻结机制；一旦资金离开可协查平台，链上单方面冻结空间非常有限。因此，公开材料重点是整理首笔损失、受害经过、关键交易记录和后续协查材料。

公开版案例链路可概括为以下方向。具体候选筛选、置信度模型和归因细节不在公开材料中展开：

```text
受害钱包
  -> 可疑归集地址
  -> 隐私化或混淆路径
  -> 后续路径整理
  -> 疑似平台或其他可协查节点
```

这个案例适合说明：隐私化路径并不等于线索完全消失，但相关判断需要专业分析和置信度控制。公开材料不展示内部判断逻辑、真实地址、交易哈希和交易所账号。

## 典型场景

钱包遭遇钓鱼攻击、私钥泄露或恶意授权后，ETH 被快速转入攻击者地址，再进入隐私化或混淆路径。后续如果出现可协查平台线索，需要尽快整理材料。

## 需要准备的材料

- 被盗钱包地址
- 首笔被盗交易哈希
- 被盗金额和时间
- 攻击者初始归集地址
- 关键路径交易记录
- 涉及金额、时间和地址
- 后续可见地址或平台线索
- 后续平台-facing 迹象
- 钓鱼网站、签名记录、钱包截图

## 公开方法边界

本公开页面不披露内部路径判断、候选筛选规则、置信度模型或归因方法。页面仅说明案例类型、公开难点和可整理的材料方向。

```text
钱包被盗或授权钓鱼经过
关键交易记录

```

## 原生币和稳定币的区别

ETH 是原生币，没有 USDT 这类发行方冻结机制。如果资金已经离开可协查平台，单纯链上冻结通常不可行。因此材料准备和对外协查节奏非常重要。

## 可交付材料

- 公开材料边界说明
- 可协查平台线索说明
- 报案辅助材料
- 交易所协查材料
