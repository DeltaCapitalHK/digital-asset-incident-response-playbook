# USDT 跨网络复杂流转追踪场景

本页面适用于诈骗或被盗资金经过多层转换、跨网络转移和后续归集的复杂流转场景。

这类案件通常用于切断普通用户对资金路径的理解。公开材料只说明可整理的证据类别，不展示具体重建方法。

## 脱敏案例摘要

一名用户被虚假“跨所稳定币套利”平台诱导，向对方提供的 EVM 链地址转入合计约 108,743 USDT。资金到账后，对方在数小时内通过多层转换和跨网络流转转移资产，最终出现疑似接近中心化平台环境的迹象。

该类案件的难点在于路径非常“碎”：普通用户看到的是一笔转账，但后续可能经过多个网络和多种交易类型。德尔泰（Delta&Capital）会整理成可复核的资金流向摘要，但不公开内部追踪方法。

公开版案例链路可概括为以下方向。具体路径识别、节点判断和归因方法不在公开材料中展开：

```text
受害钱包
  -> EVM 可疑收款地址
  -> 多层下游流转
  -> 后续路径整理
  -> 疑似平台或其他可协查节点
```

这个案例适合说明：复杂流转并不等于资金线索完全消失，但公开材料不展示真实地址、哈希、交易所账号、内部判断细节或客户身份材料。

## 典型场景

用户向某个 EVM 链地址转入 USDT 后，资金很快被：

1. 经过多层转换
2. 跨网络流转
3. 汇入疑似平台或其他可协查节点

## 需要准备的材料

- 首笔损失交易哈希
- 源链钱包地址和收款地址
- 可见路径中的关键交易记录
- 后续可见地址
- 后续关键交易记录
- 归集地址或疑似交易所地址
- 虚假平台、聊天记录和资金来源材料

## 公开方法边界

本公开页面不披露内部链上识别逻辑、节点归因标准、路径判断方法或完整追踪步骤。页面仅说明案例类型、公开难点和可整理的材料方向。

```text
Victim wallet
  -> scam recipient address on source chain
  -> intermediate movement
  -> further downstream movement
  -> downstream summary
  -> suspected platform or other reviewable node
```

## 常见错误

- 只记录源链首笔交易，不记录目标链。
- 把中间服务或合约地址误认为最终收款人。
- 只保留首笔交易，忽略后续路径摘要。
- 在没有证据时直接写“资金已进入某交易所”。

## 注意事项

复杂流转案件需要谨慎表达。公开说明应使用证据链语言：哪些已经确认，哪些是疑似，哪些需要交易所、执法机关或法律程序进一步核实。
