# USDT 跨链跳转和 DEX 多步兑换追踪场景

本页面适用于诈骗或被盗资金经过 DEX 多步兑换、包装资产、跨链桥、目标链再兑换等路径的 chain-hopping 场景。

这类案件通常用于切断普通用户对资金路径的理解，但链上仍可能通过事件日志、兑换记录和跨链记录进行重建。

## 脱敏案例摘要

一名用户被虚假“跨所稳定币套利”平台诱导，向对方提供的 EVM 链地址转入合计约 108,743 USDT。资金到账后，对方在数小时内启动 chain-hopping 路径：先通过 DEX 把 USDT 兑换为其他稳定币或包装资产，再通过跨链桥转移到 TRON，随后在目标链重新兑换和归集，最终出现疑似进入中心化交易所的链上迹象。

该类案件的难点在于路径非常“碎”：普通用户看到的是一笔 USDT 转账，但链上实际经历了 DEX 多跳兑换、桥接事件、目标链再兑换和归集。德尔泰（Delta&Capital）会逐段解码 swap 事件、跨链桥事件和目标链交易，整理成可复核的资金流向说明。

公开版案例链路可概括为以下方向。具体 DEX 路由解码、桥接对应关系和节点归因方法不在公开材料中展开：

```text
受害钱包
  -> EVM 可疑收款地址
  -> 多步兑换和跨链转移
  -> 目标链后续流转
  -> 疑似平台或其他可协查节点
```

这个案例适合说明：chain-hopping 并不等于资金消失，而是需要把每一段链上事件重新对账。公开材料不展示真实地址、哈希、交易所账号、桥协议内部细节或客户身份材料。

## 典型场景

用户向某个 EVM 链地址转入 USDT 后，资金很快被：

1. 在 DEX 中兑换成 USDC、ETH、BNB 或包装资产
2. 通过跨链桥转移到另一条链
3. 在目标链重新兑换回 USDT 或其他资产
4. 进入归集地址或交易所充值地址

## 需要准备的材料

- 首笔损失交易哈希
- 源链钱包地址和收款地址
- DEX swap 交易哈希
- 跨链桥交易哈希
- 目标链接收地址
- 目标链兑换交易哈希
- 归集地址或疑似交易所地址
- 虚假平台、聊天记录和资金来源材料

## 链上分析重点

### 1. DEX 路由解码

DEX 交易可能包含多跳兑换，例如：

```text
USDT -> USDC -> WETH -> bridge asset
```

需要查看交易事件日志，而不是只看钱包余额变化。

### 2. 跨链事件对账

跨链桥通常会在源链和目标链留下可对应的事件记录。公开材料只说明需要做跨链对账，不展开具体匹配方法。

### 3. 目标链再兑换

资金到达目标链后，可能重新兑换回 USDT 或转入更容易 off-ramp 的资产。目标链路径需要单独重建。

### 4. 交易所落点判断

如果最终资金进入交易所，需要准备进入交易所前的完整路径，而不是只提交第一笔损失交易。

## 从该案例拆出的材料方向

```text
首笔损失交易和受害经过
源链路径摘要
跨链转移摘要
目标链后续流转摘要
资金来源和所有权材料
报案/交易所/发行方协查材料
```

## 资金路径说明示例

```text
Victim wallet
  -> scam recipient address on source chain
  -> DEX swap route
  -> bridge contract
  -> destination-chain recipient
  -> destination-chain swap
  -> consolidation address
  -> suspected exchange deposit address
```

## 常见错误

- 只记录源链首笔交易，不记录目标链。
- 把 DEX 合约地址当成骗子个人地址。
- 忽略包装资产和桥资产。
- 没有把 lock/mint 或 burn/release 事件对应起来。
- 在没有证据时直接写“资金已进入某交易所”。

## 注意事项

Chain-hopping 追踪需要逐段还原。公开说明应使用证据链语言：哪些已经确认，哪些是疑似，哪些需要交易所、桥协议、执法机关或法律程序进一步核实。
