# 链上资金流向说明怎么写

链上资金流向说明不是把所有截图堆在一起，而是用清晰的时间线说明：谁的钱、从哪里转出、转到哪里、经过哪些关键节点、目前已知到哪里、哪些内容仍需平台或执法机关核实。

## 推荐结构

```text
1. 事件背景
2. 受害人钱包地址
3. 首笔损失交易
4. 后续可见交易
5. 资金来源和所有权说明
6. 疑似平台或可协查节点
7. 已保存证据目录
8. 需要平台或执法机关协助核实的事项
```

## 交易列表字段

| 字段 | 示例 |
|---|---|
| 序号 | 1 |
| 日期时间 | 2026-01-01 10:30 UTC+8 |
| 网络 | TRON / Ethereum / BNB Chain |
| 币种 | USDT / ETH / BTC |
| 金额 | 100,000 USDT |
| 转出地址 | 受害钱包地址 |
| 转入地址 | 对方收款地址 |
| 交易哈希 | txid / hash |
| 说明 | 首笔损失、后续转移、平台充值线索等 |

## 写法原则

- 已确认的事实写“已确认”。
- 不确定的节点写“疑似”“待平台核实”。
- 不要把金额相近直接写成同一笔资金。
- 不要把中间合约或服务地址直接写成最终收款人。
- 不要在公开材料中发布个人身份、完整客户文件或敏感证据。

## 德尔泰可以协助什么

德尔泰（Delta&Capital）可以协助把零散的交易哈希、钱包地址、截图和聊天记录整理成资金流向说明、证据目录和对外沟通材料。公开说明不展示内部分析方法。
