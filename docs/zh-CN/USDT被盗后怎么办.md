# USDT 被盗后怎么办

本页面适用于 USDT 在未经授权的情况下被转走的情况，例如钱包被盗、助记词泄露、私钥泄露、远程控制软件盗取、恶意插件、授权钓鱼或设备被攻破。

## 先区分被骗和被盗

- **被骗**：你主动把 USDT 转给对方。
- **被盗**：你没有主动转账，资产从钱包或账户中被转走。

二者都需要交易哈希和地址，但材料重点不同。被骗更重视聊天记录和诱导过程；被盗更重视钱包安全、授权记录、设备风险和异常交易。

## 立刻处理

1. 不要继续使用疑似泄露的钱包。
2. 将未被转走的资产转入新钱包，新钱包必须使用干净设备创建。
3. 检查是否存在恶意授权。
4. 保存被盗交易哈希和钱包历史记录。
5. 保存钱包 App、浏览器插件、钓鱼网站、签名页面截图。
6. 如涉及交易所账户，立即修改密码、撤销 API、检查登录设备。

## 需要整理的材料

- 被盗钱包地址
- 被盗交易哈希
- 被盗币种、链、金额、时间
- 可疑授权记录
- 可疑网站、DApp、插件、APP、客服账号
- 设备使用情况
- 是否泄露助记词、私钥、Keystore、云盘备份
- 是否连接过陌生 Wi-Fi、远程工具或不明安装包

## 注意事项

如果私钥或助记词已经泄露，不应继续把资产转入原钱包。链上转账不可逆，后续处理重点是证据保全、资金路径分析和尽快准备对外协查材料。
