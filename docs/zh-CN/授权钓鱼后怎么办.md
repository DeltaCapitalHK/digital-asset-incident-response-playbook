# 授权钓鱼后怎么办

授权钓鱼是指用户在不知情或被误导的情况下，给恶意合约、恶意地址或钓鱼 DApp 授权资产支配权限，导致资产被转走。

## 常见形式

- 假空投领取
- 假 NFT mint
- 假质押、假挖矿
- 假客服要求连接钱包
- 假授权恢复账户
- 钓鱼网站伪装成交易所、钱包或 DeFi 平台
- 恶意 Permit / Permit2 / approve 签名

## 立刻处理

1. 停止连接该网站。
2. 不要继续签名。
3. 用安全工具检查授权。
4. 在安全环境中撤销可疑授权。
5. 转移未受影响资产到新钱包。
6. 保存钓鱼网站、签名页面、授权哈希和被盗交易哈希。

## 要记录什么

- 授权交易哈希
- 被盗交易哈希
- 授权合约地址
- 被授权 spender 地址
- 被盗资产合约地址
- 钓鱼网站 URL
- 签名弹窗截图
- 钱包连接记录
- 资产被转走时间

## 链上分析重点

授权钓鱼案件通常需要区分：

- 用户主动转账
- 恶意合约 transferFrom
- Permit 签名后被拉走
- setApprovalForAll 导致 NFT 或资产被转走
- 多地址批量归集
- DEX 兑换或跨链转移

## 注意事项

撤销授权只能阻止后续继续被转走，不能自动追回已经转出的资产。已经被转走的资产仍需要按交易哈希和资金路径整理证据。
