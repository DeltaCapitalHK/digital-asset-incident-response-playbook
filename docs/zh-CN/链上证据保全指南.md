# 链上证据保全指南

链上证据保全的目标，是把“我被骗了”整理成可核验、可复盘、可提交的事实材料。

## 基础原则

- 交易哈希比截图更重要。
- 地址要写完整，不要只写前后几位。
- 时间要区分本地时间和 UTC 时间。
- 每一笔交易单独记录。
- 链名称不能省略。
- 不要修改、伪造或拼接证据。

## 每笔交易应记录

| 字段 | 示例 |
|---|---|
| 链 | TRON / Ethereum / BSC |
| 币种 | USDT / ETH / BTC |
| 交易哈希 | txid / transaction hash |
| 转出地址 | 受害钱包或交易所提现地址 |
| 收款地址 | 对方地址 |
| 金额 | 10000 USDT |
| 时间 | UTC + 本地时间 |
| 区块浏览器链接 | Tronscan / Etherscan / BscScan 等 |
| 备注 | 首笔损失、后续拆分、交易所归集等 |

## 截图建议

截图应包括：

- 完整地址
- 完整哈希
- 交易状态
- 区块时间
- 转出和转入地址
- 金额和币种
- 浏览器地址栏或区块浏览器页面标识

## 文件命名建议

```text
01-首笔损失交易-TRON-10000USDT.png
02-对方地址继续转出-TRON-9500USDT.png
03-虚假平台提现失败截图.png
04-聊天记录-对方要求缴纳保证金.png
```

## 证据目录建议

```text
case-evidence/

  02-wallet-addresses.txt
  03-chat-records/
  04-platform-screenshots/
  05-source-of-funds/
  06-police-report/
  07-exchange-contact/
```

## 注意事项

请勿在公开 GitHub Issue、社交平台或陌生群聊中发布包含个人身份、完整案件材料、交易所账号、联系方式、报案文件或客户文件的证据包。
