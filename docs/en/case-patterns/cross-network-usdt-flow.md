# Cross-Network USDT Flow Pattern

This page describes a scenario where USDT moves across more than one network through multiple downstream layers.

## Anonymized Case Summary

In one anonymized investment-scam scenario, a victim transferred approximately 295,590 USDT to an address on one network. The funds later moved through several downstream layers and appeared in another network environment, with signs that part of the flow may have approached a reviewable platform environment.

The main difficulty is continuity. A victim may only see the first transfer, while later movement may be spread across different networks, records, and platform environments.

The public case pattern can be summarized at a high level. Detailed linkage logic, attribution criteria, and internal review steps are not published:

```text
Victim wallet
  -> suspicious recipient
  -> multi-network downstream movement
  -> later-path review
  -> suspected platform or other reviewable node
```

Real addresses, hashes, exchange accounts, internal assessment details, and personal identifiers are not published.

## Typical Scenario

The victim sends USDT to a recipient address. The funds later move across more than one network and may approach a platform-controlled or otherwise reviewable environment.

## Materials to Prepare

- First loss transaction hash
- Victim and recipient addresses
- Key downstream transaction records
- Network and token information
- Later-path addresses, if visible
- Relevant timestamps
- Chat records, fake platform screenshots, and source-of-funds materials

## Public Method Boundary

This public page intentionally does not disclose internal linkage logic, attribution criteria, review thresholds, or platform-entry assessment methods. It only explains the case type and the categories of materials that may need to be organized.

## Public Material Categories

```text
key transaction records
multi-network path summary
later-path review summary
source-of-funds and ownership materials
reporting exchange or issuer contact materials
```

## Important Notice

Multi-network flow review should distinguish confirmed facts from suspected links. Public materials should describe the evidence category, not the internal analytical method.
