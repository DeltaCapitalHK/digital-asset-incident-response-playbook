# Ethereum Obfuscated Flow Pattern

This page describes an Ethereum incident pattern where stolen ETH or ERC20 assets enter privacy-oriented or obfuscation paths.

Public documentation should describe confirmed records, uncertainty, and limitations rather than publishing internal analysis methods.

## Anonymized Case Summary

In one anonymized wallet-theft scenario, approximately 119.78 ETH was moved out of a compromised wallet after phishing or malicious authorization activity. The attacker moved the ETH through obfuscation paths. Later downstream activity showed possible review directions and suspected platform-facing behavior.

The key difficulty is that ETH is a native asset and does not have the same issuer-freeze mechanism as some stablecoins. The practical focus is to organize the confirmed loss record, preserve evidence, and prepare exchange or legal-process materials quickly if later activity approaches a cooperative platform.

The public case pattern can be summarized at a high level. Candidate filtering, confidence modeling, and attribution details are not published:

```text
Victim wallet
  -> suspicious collection address
  -> privacy or obfuscation path
  -> later-path review
  -> suspected platform or other reviewable node
```

Real service identifiers, wallet addresses, hashes, exchange accounts, and personal identifiers are not published.

## Typical Scenario

A wallet is compromised through phishing, private-key exposure, or malicious approval. ETH is moved to attacker-controlled addresses and then through privacy-oriented or obfuscation paths before possible later consolidation or platform-facing activity.

## Materials to Prepare

- Compromised wallet address
- First theft transaction hash
- Amount and timestamp
- Attacker collection address
- Key obfuscation-path transaction records, if visible
- Later-path addresses, if visible
- Downstream consolidation or platform-facing indicators
- Phishing website, signature records, and wallet screenshots

## Public Method Boundary

This public page intentionally does not disclose internal review logic, candidate-selection rules, confidence modeling, or attribution methods. It only explains the case type and the categories of materials that may need to be organized.

## Public Material Categories

## Native Asset Consideration

ETH is a native asset and does not have the same issuer-freeze mechanism as some stablecoins. If funds leave cooperative platforms, on-chain freezing may not be available.

## Important Notice

This pattern is for documentation and analytical framing. It does not guarantee identification, freezing, or recovery.
